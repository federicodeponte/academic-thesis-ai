## üìù Documentation Standards

When adding new documentation:

1. **Choose the right category:**
   - User-facing? ‚Üí `guides/`
   - Technical/developer? ‚Üí `development/`
   - Release-related? ‚Üí `releases/`
   - Planning/audit? ‚Üí `planning/`
   - Architecture? ‚Üí `architecture/`
   - Root-level reference? ‚Üí `docs/` (this directory)

2. **Use clear filenames:**
   - ALL_CAPS_WITH_UNDERSCORES.md
   - Descriptive and concise
   - Include date if time-sensitive

3. **Add to this index:**
   - Update the appropriate section
   - Include brief description
   - Maintain alphabetical order within sections

4. **Link from relevant places:**
   - Update README.md if user-facing
   - Update CONTRIBUTING.md if developer-facing
   - Cross-link related documents
